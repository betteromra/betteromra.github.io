Tool Development and Pipeline Automation for an Open-World Fantasy Action RPG

At KRAFTON Montréal Studio, I worked on PROJECT WINDLESS, a large-scale single-player open-world action RPG built in Unreal Engine 5 and inspired by the Korean fantasy series The Bird That Drinks Tears. The project focused on real-time large-scale battles, cinematic worldbuilding, and a persistent open-world environment with multiple playable races and complex gameplay systems. My role focused on internal tool development, pipeline automation, and editor workflow optimization.

One of my main contributions was designing a standardized Blueprint inheritance architecture to reduce duplication across gameplay and editor systems. By restructuring existing tools into reusable parent-child Blueprint frameworks, I improved maintainability and enabled more consistent extension of core functionality across teams. This reduced duplicated logic and improved iteration speed for Blueprint-based workflows.

I integrated a real-time viseme-based lip-sync system into the character production pipeline, supporting both MetaHuman and custom skeletal rigs. The system generates facial animation curves directly from audio input, allowing dialogue to drive facial movement without manual keyframing.

The pipeline supported multiple audio sources, including recorded dialogue, microphone input and text-to-speech (TTS) systems used for runtime or procedural dialogue generation. This enabled characters to automatically animate speech in real time, improving iteration speed for cinematic and narrative content.

This integration reduced manual lip-sync workload and allowed early previewing of dialogue-heavy sequences during production, while remaining flexible enough for both authored cinematics and dynamic dialogue systems.

To support development of large open-world environments, I created editor tools for world navigation and location management using World Partition-based debug shapes and markers in Unreal Engine. These tools helped designers and developers quickly identify gameplay areas, testing zones, and points of interest across the world.

This improved debugging efficiency, reduced time spent navigating large levels and streamlined environment iteration across multiple teams.

I developed a suite of editor automation and workflow tools to reduce repetitive tasks and improve production efficiency within Unreal Engine.

A key tool was a Git-integrated actor revert system, allowing developers to restore modified actors directly within the editor by unloading the actor, reverting its state from version control and reloading it seamlessly in the level. This improved safety and iteration speed during level editing.

I also built a static mesh replacement tool that allowed designers to swap meshes on existing actors while preserving transforms and component setups, enabling fast visual iteration without breaking scene configuration.

For debugging and analysis, I implemented a wireframe visualization mode to toggle wireframe rendering on selected actors or entire scenes, improving visibility into geometry, collision and spatial layout.

To ensure data consistency, I added an actor validation system that automatically checked for missing components, incorrect setups and invalid configurations, helping catch issues early in the pipeline.

I also developed editor navigation and asset shortcut tools to provide quick access to frequently used Blueprints and assets, reducing friction when working across large project structures.

Overall, these tools reduced repetitive manual work and significantly improved iteration speed across the teams. The focus throughout was on building scalable, production-ready tooling to support a large AAA open-world RPG pipeline.
